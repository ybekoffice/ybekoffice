방법론

트렌드 소재 발굴 4가지 접근법

01

성장 속도 (Velocity)

절대값이 아니라 속도를 본다. 같은 댓글 500개라도 올라온 지 2일이면 신호, 6개월이면 평범.

내 데이터 한계: 댓글수가 "수집 시점 누적값"이라 정확한 속도 측정은 불완전. created_at으로 보완.
02

점유율 변화

전체 고성과 콘텐츠 중 특정 소재의 비중이 늘어나는지 추적. 3개월 전 vs 지금 비교.

내 데이터로 가능: created_at + product_type + comment_count 조합. 즉시 구현 가능.
03

신규 소재 탐지

기존에 없던 product_type이 최근에 처음 등장했는데 댓글수가 높다면 새로운 시장 신호.

내 데이터로 가능: 최근 3개월에만 등장한 product_type 필터링으로 확인.
04

외부 검색량 교차 검증

플랫폼 내부만으론 한계. 사람들이 뭘 검색하는지를 같이 봐야 수요 확인 가능.

네이버 데이터랩 API (발급 완료): 검색어 트렌드 + 쇼핑인사이트 두 가지 활용 가능.
외부 신호 연결

네이버 API 활용 방식

검색어 트렌드 API

특정 단어를 네이버에서 얼마나 검색하는지 시계열로 반환. product_type 키워드를 넣으면 검색 관심도 추이 확인 가능.

쇼핑인사이트 API

쇼핑 카테고리별 검색량 추이. 카테고리 코드 매핑이 필요하지만, 구매 의향이 있는 수요만 보여줌.

추가 분석

대본 구조 vs 소구점 — 뭐가 성과를 결정하는가?

같은 소재 안에서도 성과 차이가 나는 원인을 두 가지로 나눠 비교한다.

대본 구조 (hook_type · persuasion_arc)

같은 product_type 안에서 hook_type이 다를 때 댓글수 차이 → 구조가 얼마나 영향을 주는지

소구점 (selling_point)

같은 hook_type 안에서 selling_point 패턴이 다를 때 댓글수 차이 → 무엇을 어필하느냐의 영향
